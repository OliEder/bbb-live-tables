== EinfГјhrung und Ziele

=== Aufgabenstellung

BBB Live Tables ist ein WordPress-Plugin, das Liga-Tabellen und Turnier-Brackets des Deutschen Basketball Bundes (DBB) live auf einer Vereinswebsite darstellt. Die Daten werden Гјber die Г¶ffentliche REST API von https://www.basketball-bund.net[basketball-bund.net] (BBB) bezogen.

Das Plugin wurde fГјr den Basketballverein **Fibalon Baskets Neumarkt** (BBB Club-ID: 4468) entwickelt, ist aber fГјr jeden deutschen Basketballverein nutzbar.

=== Wesentliche Features

* **Liga-Tabellen** mit konfigurierbaren Spalten (Desktop/Mobil), Team-Logos, berechneten Statistiken (ГҳK, ГҳG, K:G, GB)
* **Turnier-Brackets** fГјr KO-Wettbewerbe (Single Elimination) und Playoff-Serien (Best-of-N)
* **Drei Ausgabe-KanГӨle:** WordPress Shortcode, Gutenberg Block, Goodlayers Page Builder Element
* **Standalone-Betrieb** ohne weitere Plugin-AbhГӨngigkeiten
* **Optionale Integration** mit `bbb-sportspress-sync` via WordPress Filter Hooks (lose Kopplung)

== LГ¶sungsstrategie

=== Kernentscheidungen

[cols="1,3",options="header"]
|===
| Entscheidung | BegrГјndung

| **Standalone ohne SportsPress**
| Viele Vereinswebsites nutzen kein SportsPress. Tabellen + Brackets sind der hГӨufigste Bedarf.

| **Live-Rendering statt lokale Kopie**
| DSGVO: Keine persistente Speicherung fremder Vereinsdaten. Tabelle wird bei jedem (Cache-Miss-)Aufruf von der BBB-API geladen.

| **Transient-Caching**
| WordPress Transients als Cache-Layer. Default 15 Min (Tabelle) / 60 Min (Bracket). Invalidierung Гјber Admin.

| **Filter Hooks statt harte AbhГӨngigkeit**
| 6 WordPress Filter Hooks ermГ¶glichen dem Sync-Plugin, reichere Daten (SP-Logos, SP-Links, SP-Farben) einzuschleusen, ohne dass das Standalone-Plugin davon wissen muss.

| **Drei Ausgabe-KanГӨle**
| Shortcode (universell), Gutenberg Block (moderner Editor), Goodlayers Element (BigSlam Page Builder) вҖ“ gleiche Render-Logik, verschiedene Eingabe-UIs.

| **CSS inline statt externes Stylesheet**
| Vermeidet zusГӨtzlichen HTTP-Request. CSS wird einmal pro Pageload injected (`$css_injected` Flag). Theme-Farben dynamisch aus Optionen/Filtern.
|===

==== BBB_Api_Client

**Verantwortung:** HTTP-Kommunikation mit der BBB REST API.

* Stripped-Variante: Nur Endpoints fГјr Tabelle, Bracket und Team-Discovery

== Architekturentscheidungen

=== ADR-001: Standalone statt SportsPress-Erweiterung

**Kontext:** UrsprГјnglich war die Tabellen-/Bracket-FunktionalitГӨt Teil von `bbb-sportspress-sync`. Viele Vereinswebsites nutzen aber kein SportsPress.

**Entscheidung:** Extraktion als eigenstГӨndiges Plugin mit Filter-Hook-Integration.

**Konsequenzen:**

* (+) Breitere Nutzbarkeit ohne SP-AbhГӨngigkeit
* (+) DSGVO-konformer (kein lokaler SP-Datenbestand nГ¶tig fГјr Tabellen)
* (+) UnabhГӨngige Versionierung
* (вҲ’) Duplizierung des API-Clients (stripped vs. full)
* (вҲ’) Conditional Loading Logik in beiden Plugins

=== ADR-002: Filter Hooks statt Events oder Service Locator

**Kontext:** Die Integration zwischen Standalone und Sync-Plugin musste lose gekoppelt sein.

**Entscheidung:** WordPress `apply_filters()` mit sinnvollen Standalone-Defaults.

**Konsequenzen:**

* (+) Null-Konfiguration: Beide Plugins erkennen sich automatisch
* (+) WordPress-nativer Ansatz, keine Custom-Infrastruktur
* (+) Drittanbieter kГ¶nnen eigene Filter registrieren
* (вҲ’) Keine Typisierung der Filter-RГјckgabewerte

=== ADR-003: Inline-CSS statt externes Stylesheet

**Kontext:** Theme-Farben sind dynamisch (Optionen, Goodlayers, SP-Frontend-Farben).

**Entscheidung:** CSS wird inline generiert mit dynamischen Farben aus der Fallback-Kette.

**Konsequenzen:**

* (+) Kein extra HTTP-Request
* (+) Farben passen sich automatisch an Theme/Optionen an
* (+) Einmal-Injection via `$css_injected` Flag
* (вҲ’) Nicht cache-bar als separates Asset

=== ADR-004: Goodlayers-Registrierung nur ohne Sync-Plugin

**Kontext:** Beide Plugins registrieren Goodlayers-Elemente mit unterschiedlicher KomplexitГӨt (Standalone: manuell, Sync: SP-basiertes Liga-Dropdown).

**Entscheidung:** Standalone prГјft `defined('BBB_SYNC_VERSION')` und Гјberspringt bei aktivem Sync-Plugin.

**Konsequenzen:**

* (+) Keine doppelte Registrierung
* (+) Sync-Plugin liefert automatisch reichere UI
* (вҲ’) Timing-AbhГӨngigkeit (`plugins_loaded` Priority 10 vs 20)
