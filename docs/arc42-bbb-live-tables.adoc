== Einführung und Ziele

=== Aufgabenstellung

BBB Live Tables ist ein WordPress-Plugin, das Liga-Tabellen und Turnier-Brackets des Deutschen Basketball Bundes (DBB) live auf einer Vereinswebsite darstellt. Die Daten werden über die öffentliche REST API von https://www.basketball-bund.net[basketball-bund.net] (BBB) bezogen.

Das Plugin wurde für den Basketballverein **Fibalon Baskets Neumarkt** (BBB Club-ID: 1234) entwickelt, ist aber für jeden deutschen Basketballverein nutzbar.

=== Wesentliche Features

* **Liga-Tabellen** mit konfigurierbaren Spalten (Desktop/Mobil), Team-Logos, berechneten Statistiken (ØK, ØG, K:G, GB)
* **Turnier-Brackets** für KO-Wettbewerbe (Single Elimination) und Playoff-Serien (Best-of-N)
* **Drei Ausgabe-Kanäle:** WordPress Shortcode, Gutenberg Block, Goodlayers Page Builder Element
* **Standalone-Betrieb** ohne weitere Plugin-Abhängigkeiten
* **Optionale Integration** mit `bbb-sportspress-sync` via WordPress Filter Hooks (lose Kopplung)

=== Qualitätsziele

[cols="1,3,1",options="header"]
|===
| Priorität | Qualitätsziel | Maßnahme

| 1
| **DSGVO-Konformität / Datensparsamkeit**
| Keine persistente Speicherung fremder Vereinsdaten. Nur Transient-Caching.

| 2
| **Wartbarkeit / Unabhängigkeit**
| Kein SportsPress, kein externes Build-System, reines PHP + Vanilla JS.

| 3
| **Performance**
| Transient-Caching aller API-Responses. Keine API-Calls bei gecachtem Content.

| 4
| **Theme-Kompatibilität**
| Getestet mit BigSlam/Goodlayers. CSS mit `!important` wo nötig (Theme-Override).
|===

=== Stakeholder

[cols="1,2",options="header"]
|===
| Stakeholder | Erwartung

| Vereinstrainer / Webmaster
| Aktuelle Tabellen + Brackets auf der Website ohne manuellen Pflegeaufwand

| Websitebesucher
| Schnelle, korrekte, responsive Darstellung auf Desktop und Mobil

| Datenschutzbeauftragter
| Keine unnötige Speicherung personenbezogener Daten (Spielernamen etc.)

| Plugin-Entwickler
| Saubere Schnittstelle zur Erweiterung via Filter Hooks
|===

== Lösungsstrategie

=== Kernentscheidungen

[cols="1,3",options="header"]
|===
| Entscheidung | Begründung

| **Standalone ohne SportsPress**
| Viele Vereinswebsites nutzen kein SportsPress. Tabellen + Brackets sind der häufigste Bedarf.

| **Live-Rendering statt lokale Kopie**
| DSGVO: Keine persistente Speicherung fremder Vereinsdaten. Tabelle wird bei jedem (Cache-Miss-)Aufruf von der BBB-API geladen.

| **Transient-Caching**
| WordPress Transients als Cache-Layer. Default 15 Min (Tabelle) / 60 Min (Bracket). Invalidierung über Admin.

| **Filter Hooks statt harte Abhängigkeit**
| 6 WordPress Filter Hooks ermöglichen dem Sync-Plugin, reichere Daten (SP-Logos, SP-Links, SP-Farben) einzuschleusen, ohne dass das Standalone-Plugin davon wissen muss.

| **Drei Ausgabe-Kanäle**
| Shortcode (universell), Gutenberg Block (moderner Editor), Goodlayers Element (BigSlam Page Builder) – gleiche Render-Logik, verschiedene Eingabe-UIs.

| **CSS inline statt externes Stylesheet**
| Vermeidet zusätzlichen HTTP-Request. CSS wird einmal pro Pageload injected (`$css_injected` Flag). Theme-Farben dynamisch aus Optionen/Filtern.
|===

=== Plugin-Ladereihenfolge

[source]
----
plugins_loaded Priority 10:  bbb-sportspress-sync (definiert BBB_SYNC_VERSION)
plugins_loaded Priority 20:  bbb-live-tables      (prüft BBB_SYNC_VERSION + BBB_TABLES_VERSION)
----

Wenn beide aktiv: Sync-Plugin registriert Filter-Callbacks → Standalone nutzt automatisch SP-Daten.
Wenn nur Standalone: Eigene Defaults (BBB-API-Logos, keine Team-Links).

=== Ebene 2: Bausteinbeschreibungen

==== BBB_Api_Client

**Verantwortung:** HTTP-Kommunikation mit der BBB REST API.

* Stripped-Variante: Nur Endpoints für Tabelle, Bracket und Team-Discovery
* Rate Limiting (1 req/s Throttle)
* Error Handling → `WP_Error`
* Logging nach `error_log` (nur bei `WP_DEBUG`)
* Conditional Loading: Wird _nicht_ geladen wenn das Sync-Plugin seinen eigenen (erweiterten) Client mitbringt (`class_exists('BBB_Api_Client')`)

==== BBB_Live_Table

**Verantwortung:** Rendering der Liga-Tabelle in allen drei Ausgabe-Kanälen.

Kernfunktionen:

* Shortcode `[bbb_table]` mit 10 Parametern
* Gutenberg Block `bbb/league-table` mit ServerSideRender
* Spalten-Konfiguration (Desktop/Mobil getrennt)
* Berechnete Spalten: `ppg` (ØK), `oppg` (ØG), `korbRatio` (K:G), `gb` (Games Behind)
* Team-Display-Modi: `full`, `short`, `logo`, `nameShort`
* Responsive CSS (Spalten-Visibility per Breakpoint)
* Theme-Farben via Filter oder Plugin-Settings
* Eigene-Teams-Highlighting via Club-ID
* Transient-Cache mit konfigurierbarer TTL

=== Caching-Strategie

[cols="2,1,2",options="header"]
|===
| Daten | TTL (Default) | Transient-Key

| Liga-Tabelle
| 900s (15 Min)
| `bbb_table_{liga_id}`

| Turnier-Bracket
| 3600s (1 Std)
| `bbb_bracket_{liga_id}`

| Club-Teams (Discovery)
| 86400s (24 Std)
| `bbb_club_teams_{club_id}`

| Club-Ligen (Discovery)
| 86400s (24 Std)

Invalidierung:

* Admin-Button "Cache leeren"
* Sync-Plugin invalidiert bei erfolgreichem Sync
* Gutenberg-Editor: Cache auf max 60s reduziert (Preview-Modus)

=== Filter-Hook-Architektur

Sechs Filter Hooks ermöglichen die lose Kopplung mit `bbb-sportspress-sync`:

[source,php]
----
// Logo: Filter → BBB-API-Fallback
$url = apply_filters('bbb_table_team_logo_url', '', $team_pid);
if (!$url) {
    $url = "https://www.basketball-bund.net/media/team/{$team_pid}/logo";
}

// Team-Link: Filter → leer (kein Link)
$url = apply_filters('bbb_table_team_url', '', $team_pid);

// Event-Link: Filter → leer (kein Link)
$url = apply_filters('bbb_table_event_url', '', $match_id);

// Farben: Filter → Plugin-Settings → Goodlayers Theme
$colors = apply_filters('bbb_table_theme_colors', $defaults);

// Eigene Teams: Filter → API Auto-Discovery → manuelle PIDs
$ids = apply_filters('bbb_table_own_team_ids', [], $club_id);

// Liga-Optionen für Editor: Filter → leeres Array
$options = apply_filters('bbb_table_liga_options', [], $type_filter);
----

[cols="2,3,3",options="header"]
|===
| Filter | Ohne Sync-Plugin | Mit Sync-Plugin

| `bbb_table_team_logo_url`
| BBB Media URL
| SportsPress Featured Image (sportspress-fit-icon)

| `bbb_table_team_url`
| Leer (Plain Text)
| SP Team Post Permalink

| `bbb_table_event_url`
| Leer (kein Link)
| SP Event Post Permalink

| `bbb_table_theme_colors`
| Plugin-Settings + Goodlayers Fallback
| SP Frontend CSS Colors + Goodlayers

| `bbb_table_own_team_ids`
| API Auto-Discovery (24h Cache)
| SP-Team-Posts mit `_bbb_is_own_team` Meta

| `bbb_table_liga_options`
| Leeres Array (manuelle Liga-ID)
| `sp_league` Taxonomy Terms mit `_bbb_liga_id`
|===

=== Eigene-Teams-Erkennung (Fallback-Kette)

. Filter `bbb_table_own_team_ids` (Sync-Plugin liefert SP-basierte IDs)
. API Auto-Discovery via `get_club_team_ids()` (24h Transient-Cache)
. Manuelle Liste aus `bbb_tables_own_team_pids` Option

=== BigSlam-CSS-Kompatibilität

Das BigSlam-Theme überschreibt globale `<th>`-Styles mit dunklem Hintergrund. Das Plugin reagiert mit:

* `tbody td, tbody th { background:transparent!important; color:#222!important }`
* Zebrastreifen auf 90% Aufhellung (statt 97% – deutlicher sichtbar)
* Hover, Own-Team-Highlighting etc. mit `!important` für `td` UND `th`

=== DSGVO / Datensparsamkeit

* **Keine persistente Speicherung** fremder Vereinsdaten
* Nur WordPress Transients als Cache (automatisch ablaufend)
* Team-Logos werden direkt von der BBB-API geladen (kein lokaler Download)
* Keine Cookies, kein Tracking, kein Analytics
* Keine Spielernamen in Tabellen (nur Teamnamen)

== Architekturentscheidungen

=== ADR-001: Standalone statt SportsPress-Erweiterung

**Kontext:** Ursprünglich war die Tabellen-/Bracket-Funktionalität Teil von `bbb-sportspress-sync`. Viele Vereinswebsites nutzen aber kein SportsPress.

**Entscheidung:** Extraktion als eigenständiges Plugin mit Filter-Hook-Integration.

**Konsequenzen:**

* (+) Breitere Nutzbarkeit ohne SP-Abhängigkeit
* (+) DSGVO-konformer (kein lokaler SP-Datenbestand nötig für Tabellen)
* (+) Unabhängige Versionierung
* (−) Duplizierung des API-Clients (stripped vs. full)
* (−) Conditional Loading Logik in beiden Plugins

=== ADR-002: Filter Hooks statt Events oder Service Locator

**Kontext:** Die Integration zwischen Standalone und Sync-Plugin musste lose gekoppelt sein.

**Entscheidung:** WordPress `apply_filters()` mit sinnvollen Standalone-Defaults.

**Konsequenzen:**

* (+) Null-Konfiguration: Beide Plugins erkennen sich automatisch
* (+) WordPress-nativer Ansatz, keine Custom-Infrastruktur
* (+) Drittanbieter können eigene Filter registrieren
* (−) Keine Typisierung der Filter-Rückgabewerte

=== ADR-003: Inline-CSS statt externes Stylesheet

**Kontext:** Theme-Farben sind dynamisch (Optionen, Goodlayers, SP-Frontend-Farben).

**Entscheidung:** CSS wird inline generiert mit dynamischen Farben aus der Fallback-Kette.

**Konsequenzen:**

* (+) Kein extra HTTP-Request
* (+) Farben passen sich automatisch an Theme/Optionen an
* (+) Einmal-Injection via `$css_injected` Flag
* (−) Nicht cache-bar als separates Asset

=== ADR-004: Goodlayers-Registrierung nur ohne Sync-Plugin

**Kontext:** Beide Plugins registrieren Goodlayers-Elemente mit unterschiedlicher Komplexität (Standalone: manuell, Sync: SP-basiertes Liga-Dropdown).

**Entscheidung:** Standalone prüft `defined('BBB_SYNC_VERSION')` und überspringt bei aktivem Sync-Plugin.

**Konsequenzen:**

* (+) Keine doppelte Registrierung
* (+) Sync-Plugin liefert automatisch reichere UI
* (−) Timing-Abhängigkeit (`plugins_loaded` Priority 10 vs 20)

== Glossar

[cols="2,4",options="header"]
|===
| Begriff | Bedeutung

| BBB
| basketball-bund.net – Offizielle Website des Deutschen Basketball Bundes (DBB)

| Liga-ID
| Eindeutige Kennung einer Liga/eines Wettbewerbs in der BBB-API

| Team Permanent-ID (PID)
| Mannschafts-ID die über Saisons hinweg gleich bleibt (im Gegensatz zur saisonalen Team-ID)

| Club-ID
| Vereins-ID in der BBB-API (z.B. 1234)

| SP / SportsPress
| WordPress-Plugin für Sportvereinsverwaltung (Teams, Events, Spieler, Tabellen)

| Goodlayers / gdlr_core
| Page Builder Framework, genutzt vom BigSlam WordPress-Theme

| Transient
| WordPress-eigener Key-Value-Cache mit automatischer Ablaufzeit

| Games Behind (GB)
| Statistik: Rückstand eines Teams auf den Tabellenführer in Siegen

| KO / Single Elimination
| Turnierformat: Verlierer scheidet direkt aus

| Playoff / Best-of-N
| Turnierformat: Serie aus N Spielen, wer zuerst Mehrheit gewinnt
